Frequently Asked Questions

What is the difference between var, let, and const?
In JavaScript, `var`, `let`, and `const` differ in scope, re-declaration, initialization, and hoisting.
=> **Scope:** `var` has function scope, while `let` and `const` have block scope.
=>**Re-declaration & Updates:** `var` can be updated and re-declared. `let` can be updated but not re-declared. `const` cannot be updated or re-declared.
=>**Initialization:** `var` and `let` can be declared without initialization, but `const` must be initialized at declaration.
=>**Access before Initialization:** `var` is accessible before initialization (default: `undefined`), whereas `let` and `const` throw a `ReferenceError`. =>**Hoisting:** `var` is hoisted and initialized with `undefined`, while `let` and `const` are hoisted but not initialized, causing an error if accessed before declaration.
Use `let` for variables that change and `const` for constants. Avoid `var` due to its functional scope and hoisting behavior.
What is the difference between map(), forEach(), and filter()?
In JavaScript, map(), forEach(), and filter() are array methods with distinct purposes:
• map(): Creates a new array by applying a function to each element. It does not modify the original array and is useful when you need a transformed array.
• forEach(): Executes a function on each array element but does not return a new array. It is used for operations like logging or modifying elements outside the loop.
• filter(): Creates a new array containing only elements that pass a given condition (return true). It does not modify the original array and is useful for extracting specific elements.
Key Differences:
=>'map()' and 'filter()' produce new arrays, whereas 'forEach()' does not.
=>'map()' changes items, 'filter()' selects elements, and 'forEach()' executes side effects.
=>'map()' and 'filter()' can be chained, but 'forEach()' cannot.
Use map() for transformation, filter() for selection, and forEach() for iteration without returning a new array.
Explain arrow functions and how they are different from regular functions?
Arrow functions, which were introduced in ES6, provide a succinct approach to construct anonymous functions in JavaScript. They employ the '=>' syntax and have an implied return for single-expression functions.
k**Differences from Regular Functions:**
1. **'this' Binding:** Regular functions have their own 'this' that is determined by how they are called.
- Arrow functions inherit 'this' from their context (lexical 'this').
2. Syntax:
- Regular functions require the 'function' keyword.
- Arrow functions are shorter.
'''JS const add = (a, b) => a + b; // Implicit return '''
3. **Arguments Object:** Regular functions include a 'arguments' object.
- Arrow functions don't; instead, they use rest parameters ('...args').
4. **Usage:** - Use arrow functions for callbacks, array methods, and to preserve 'this'.
- Use ordinary functions for object methods and where 'this' must be dynamic.
How does JavaScript Promises work?
JavaScript **Promises** help manage asynchronous operations, preventing **callback hell**. A **Promise** represents a value that may be available now, in the future, or never.
**States of a Promise:**
1. **Pending** - initial state, waiting for completion.
2. **Fulfilled** – The operation was successful (`resolve()` called).
3. **Rejected** – The operation failed (`reject()` called).
4. **Settled** – The Promise has either been fulfilled or rejected.
### **Creating and Handling a Promise:**
JavaScript
let promise = new Promise((resolve, reject) => {
let success = true;
success ? resolve("Success!") : reject("Error!");
});
promise
.then(result => console.log(result))
.catch(error => console.error(error))
.finally(() => console.log("Done"));
```
Promises improve readability, making asynchronous code cleaner and easier to handle, especially when combined with **async/await**.
How closures work in JavaScript?
A **closure** in JavaScript is a function that retains access to its parent’s variables even after the parent function has executed. This happens because JavaScript functions remember the scope in which they were created.
**How Closures Work:**
When a function is defined inside another function, it forms a closure by "closing over" the variables from its outer scope.
**Example:**
```js
function outerFunction(outerVariable) {
return function innerFunction(innerVariable) {
console.log(`Outer: ${outerVariable}, Inner: ${innerVariable}`);
}; }
const closureExample = outerFunction("Hello");
closureExample("World"); // Output: Outer: Hello, Inner: World
```
Here, `innerFunction` retains access to `outerVariable` even after `outerFunction` has finished executing.
**Uses of Closures:**
- Data privacy (creating private variables)
- Function factories
- Memoization (caching results)
- Event handling
Closures help in writing efficient, modular, and maintainable code.
